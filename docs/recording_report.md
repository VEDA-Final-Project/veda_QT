# 사진 및 영상 저장 기능 구현 보고서

1.	목 적
-	parkingService의 입출차 이벤트, 관리자의 조작패드 액션 등에서 사용될 범용적인 영상, 캡쳐 기능을 구현하고자 함.

2.	세부 구현 기능
가)	사진 캡쳐 및 영상 녹화
-	외부 이벤트에 따라 사진을 캡쳐하고 영상을 저장한다.
나)	구간 영상 저장
-	이벤트 발생 시점 일정시간(5초) 전후의 영상이 저장된다.
다)	과거 영상 보기
-	일정 시간(1시간) 동안의 과거 시점 영상을 탐색 및 재생할 수 있다.
라)	DB CRUD(Create, Read, Update, Delete) 기능
-	녹화된 동영상과 캡쳐된 사진을 전용 탭에서 CRUD 할 수 있다.

3.	함수 동작 설명
가)	수동 캡쳐 및 녹화 제어
- **`onCaptureManual()`**: 호출 시점의 선택된 채널 버퍼에서 가장 최신 프레임을 JPG 파일로 스냅샷 저장합니다.
- **`onRecordManualToggled(bool checked)`**: 사용자가 버튼을 수동으로 조작할 때 호출되며, `true` 시 녹화 시작, `false` 시 녹화를 종료하고 MP4 파일로 병합하여 저장합니다.

나)	이벤트 기반 자동 구간 저장
- **`eventRecordRequested(description, preSec, postSec)`**: 입출차 등 특정 이벤트 발생 시 트리거되는 신호입니다.
- **동작 방식**: 클릭 시점 이전(`preSec`) 프레임은 버퍼에서 즉시 확보하고, 이후(`postSec`) 프레임은 타이머를 통해 대기했다가 확보하여 하나의 MP4 영상으로 자동 생성합니다.

다)	상시 녹화 데이터 보존 및 삭제
- **`onCleanupTimeout()`**: 설정된 보존 기간(예: 1시간)이 경과한 오래된 미디어 파일과 DB 레코드를 주기적으로 자동 삭제하여 디스크 공간을 관리합니다.

라)	DB 연동 및 데이터 관리
- **`MediaRepository`**: 저장된 모든 사진과 영상의 메타데이터(채널, 파일 경로, 생성 시간 등)를 SQLite DB에 저장하고 관리하며, 녹화조회 탭에서의 CRUD 기능을 지원합니다.

4.	사용 예시
가)	사용자는 Qt UI 버튼 및 조작패드를 눌러 선택한 카메라의 사진을 캡쳐하고 동영상으로 영상을 저장한다.
나)	입출차가 발생 이벤트가 발생하면 `eventRecordRequested`를 통해 10초간의 영상이 자동으로 저장된다.
다)	사용자는 정해진 기간(1시간) 이내의 과거 영상을 돌려본다.
라)	“메뉴>녹화조회” 탭에서 저장된 사진과 영상을 조회하고 삭제한다.
