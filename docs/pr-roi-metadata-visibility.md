# PR: ROI는 주차 판정에만 사용하고 메타데이터는 항상 표시

## 배경

기존에는 ROI가 활성화되면 화면 렌더러가 ROI와 겹치지 않는 객체 메타데이터 박스/라벨을 그리지 않았습니다.

이 때문에 사용자는:

- ROI를 생성한 직후 기존에 보이던 차량/사람/번호판 메타데이터가 사라진 것처럼 느끼고
- 실제 인식이 멈춘 것인지, 단순히 표시만 안 되는 것인지 구분하기 어려웠으며
- ROI가 주차 판정 영역인지, 화면 표시 필터인지 혼동할 수 있었습니다

이번 변경은 ROI의 역할을 `주차 판정용 영역`으로 한정하고, 화면 메타데이터 오버레이는 ROI 유무와 관계없이 항상 표시되도록 정리하는 데 목적이 있습니다.

## 주요 변경 사항

### 1. ROI 기반 객체 오버레이 필터 제거

- `VideoFrameRenderer::compose(...)`에서 ROI와 객체 박스 교차 여부를 검사하던 분기를 제거
- ROI가 활성화되어 있어도 객체 메타데이터 박스/라벨을 더 이상 숨기지 않음
- 차량, 사람, 번호판 등 기존 메타데이터 오버레이가 ROI 밖에서도 계속 표시됨

### 2. ROI 시각화는 그대로 유지

- ROI 폴리곤 표시 유지
- 점유 상태에 따른 색상 표시 유지
- 존 라벨 및 점유 라벨(`Parked`) 표시 유지

### 3. 주차 판정 로직은 변경하지 않음

- ROI는 계속 `VehicleTracker`와 `ParkingService`의 점유 판정에 사용
- 입차/출차 판단, 점유 존 계산, 히스토리 안정화 로직은 그대로 유지
- 이번 변경은 표시 계층만 수정하고, 메타데이터 수집/주차 DB 반영 흐름은 건드리지 않음

### 4. 오해 소지가 있는 렌더링 주석 정리

- ROI가 화면 메타데이터 필터처럼 보이게 하던 주석과 불필요한 보조 계산을 제거
- 코드가 실제 동작과 더 잘 맞도록 정리

## 영향 범위

- CCTV 화면의 객체 메타데이터 오버레이 표시 방식
- ROI 생성 이후 사용자 체감 동작

주차 입출차 판정 규칙, DB 스키마, OCR 파이프라인은 변경하지 않습니다.

## 관련 파일

- `src/presentation/widgets/videoframerenderer.cpp`

## 확인 방법

1. ROI가 없는 상태에서 기존처럼 객체 메타데이터가 정상 표시되는지 확인
2. ROI를 하나 이상 생성한 뒤에도 ROI 밖 차량/사람/번호판 메타데이터가 계속 보이는지 확인
3. ROI 안 차량은 기존처럼 점유 상태와 존 라벨이 정상 표시되는지 확인
4. ROI 생성/삭제 직후 메타데이터 오버레이가 사라지거나 깜빡이며 숨겨지지 않는지 확인
5. ROI 밖 객체가 보이더라도 실제 주차 점유/입출차 판정은 기존과 동일하게 동작하는지 확인

## 리뷰 포인트

- 표시 계층에서만 변경이 일어나고 주차 판정 로직은 그대로 유지되는지
- ROI 시각화는 유지하면서 객체 메타데이터 필터만 제거되었는지
- ROI 생성 후 메타데이터가 사라지던 기존 UX 혼선이 해소되는지

## 비고

- 이 변경은 메타데이터를 `더 많이 표시`하도록 하는 UX 수정이며, 인식 모델이나 메타데이터 입력 파이프라인을 바꾸는 작업은 아닙니다.
- 현재 환경에서는 전체 빌드를 끝까지 확인하지 못해 컴파일 검증은 별도로 필요합니다.
