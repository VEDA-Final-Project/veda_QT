# PR: Raspberry Pi 인증 서버 기반 2단계 로그인 도입

## 배경

기존 로그인 화면은 고정 테스트 계정(`admin` / `1234`) 비교 방식으로만 동작했습니다.  
실제 운영 환경에서 사용할 수 있도록 Raspberry Pi 인증 서버와 연동되는 2단계 로그인 흐름을 추가했습니다.

## 주요 변경 사항

### 1. 로그인 UI를 2단계 흐름으로 변경

- 1차: 아이디/비밀번호 입력 후 서버에 로그인 요청
- 2차: 서버 응답이 성공이면 OTP 입력 단계로 전환
- OTP 단계에서 계정 마스킹 문구와 뒤로가기 버튼 제공
- 진행 상태, 성공, 실패 메시지를 단계별로 구분해 표시

### 2. `RpiAuthClient` 신규 추가

- Raspberry Pi 인증 서버와 TCP 소켓으로 통신하는 인증 클라이언트 추가
- 연결 타임아웃 / 요청 타임아웃 처리 추가
- 길이 프리픽스 + JSON 기반 메시지 송수신 처리
- 로그인 성공, OTP 대기, 인증 성공, 프로토콜 오류, 서버 연결 실패 케이스를 분리 처리

### 3. 설정 파일에 인증 서버 정보 추가

- `config/settings.json`에 `auth` 섹션 추가
- 설정 항목:
  - `host`
  - `port`
  - `connectTimeoutMs`
  - `requestTimeoutMs`
- `Config`에서 위 값을 읽을 수 있도록 getter 추가

### 4. 빌드 대상에 인증 클라이언트 등록

- `CMakeLists.txt`에 `src/infrastructure/rpi/rpiauthclient.cpp`, `src/infrastructure/rpi/rpiauthclient.h` 추가

## 기대 효과

- 테스트용 하드코딩 로그인에서 실제 인증 서버 기반 로그인으로 전환할 수 있습니다.
- 비밀번호 검증과 OTP 검증을 분리해 인증 절차를 명확하게 구성했습니다.
- 서버 미응답, 잘못된 응답, 잘못된 OTP 등 실패 상황에서 사용자 피드백이 개선됩니다.

## 확인 방법

1. 앱 실행 후 로그인 페이지 진입
2. 아이디/비밀번호 입력 후 `다음` 클릭
3. 인증 서버가 정상 응답하면 OTP 입력 화면으로 전환되는지 확인
4. 잘못된 계정 입력 시 에러 메시지와 비밀번호 초기화 동작 확인
5. 잘못된 OTP 입력 시 OTP만 초기화되고 재입력 가능한지 확인
6. 서버 미연결 또는 타임아웃 상황에서 안내 문구가 노출되는지 확인
7. OTP 단계에서 `뒤로가기` 클릭 시 초기 로그인 단계로 복귀하는지 확인

## 영향 범위

- 로그인 화면 UI/상태 전이
- 설정 로딩(`Config`)
- Raspberry Pi 인증 서버 통신 레이어
- 빌드 설정

## 리뷰 포인트

- 인증 서버 프로토콜(`ready`, `login_ok`, `auth_ok`, `error`)과 현재 클라이언트 구현이 정확히 일치하는지
- 로그인 실패/OTP 실패/타임아웃 시 세션 초기화가 의도대로 동작하는지
- `settings.json`의 인증 서버 주소/포트가 배포 환경에 맞게 관리되는지
