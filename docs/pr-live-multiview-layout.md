# PR: 카메라 라이브 뷰를 1/2/4 선택 기반 자동 레이아웃으로 전환

## 배경

기존 CCTV 메인 화면은 중앙 영상 영역이 4슬롯 고정 구조였습니다.

- `VideoWidget` 4개가 항상 2x2로 배치됨
- 채널 카드와 중앙 영상 슬롯이 사실상 1:1로 고정됨
- 1채널만 보고 싶어도 나머지 슬롯 구조를 그대로 유지해야 했음

운영 관점에서는 선택한 채널 수에 따라 화면이 더 자연스럽게 바뀌는 편이 사용성이 좋기 때문에, 라이브 뷰를 선택 기반 자동 레이아웃으로 전환했습니다.

## 주요 변경 사항

### 1. 라이브 선택 상태를 단일 선택에서 다중 선택으로 변경

- 기존 `m_selectedChannelIndex` 중심 구조를 `m_selectedChannelIndices` 기반으로 확장
- 채널 카드는 단순 전환이 아니라 선택/해제 토글로 동작
- 마지막 1개 선택은 유지되도록 하여 선택 0개 상태는 막음
- 선택 순서를 유지해 화면 슬롯 배치 순서에 반영

### 2. 선택된 채널 수에 따라 라이브 레이아웃 자동 전환

- 선택 1개: 단일 대형 뷰
- 선택 2개: 2분할 대형 뷰
- 선택 3개 이상: 4분할 뷰

별도의 모드 버튼을 두지 않고, 채널 카드 선택 상태만으로 레이아웃이 자동으로 바뀌도록 구성했습니다.

### 3. 중앙 영상 슬롯과 실제 카메라 채널의 고정 결합 제거

- 기존에는 카드 인덱스와 영상 슬롯 인덱스가 사실상 같은 개념이었음
- 변경 후에는 선택된 카메라 채널 목록을 기준으로 visible 슬롯에 동적으로 재배치
- `CameraChannelRuntime`는 고정 슬롯 역할만 담당하고, 어떤 카메라를 보여줄지는 컨트롤러가 재할당

이 구조로 인해 같은 4개의 `VideoWidget`를 재사용하면서도 1/2/4 레이아웃을 모두 지원할 수 있게 되었습니다.

### 4. 비선택 채널 스트림은 백그라운드 유지

- 이번 변경은 UI 레이아웃 개선이 목적
- `CameraSource` 4개는 계속 유지
- 선택되지 않은 채널도 썸네일, 상태 표시, 녹화/버퍼 흐름은 기존처럼 유지

즉, 화면에 크게 보이는 채널만 바뀌고 스트림 자체를 끊지는 않습니다.

### 5. ROI/부가 패널 동작을 실제 채널 기준으로 정리

- ROI 대상 콤보는 계속 `Ch1 ~ Ch4` 실제 채널 기준으로 유지
- ROI 편집 대상은 화면의 몇 번째 슬롯인지가 아니라 실제 카메라 채널 기준으로 동작
- ROI 작업 시 해당 채널이 현재 화면에 없으면 자동으로 선택 집합에 포함시켜 표시
- DB 패널의 zone 데이터 공급도 현재 visible 슬롯이 아니라 실제 채널 기준으로 유지

## UI 동작 요약

### 채널 카드

- 클릭 시 선택/해제 토글
- 선택된 카드는 활성 스타일 유지
- 비선택 카드도 썸네일과 상태 점은 계속 갱신

### 중앙 라이브 화면

- 선택 수에 맞춰 1분할, 2분할, 4분할로 자동 재배치
- 사용하지 않는 슬롯은 숨기고 consumer 연결 해제
- 재선택 시 해당 채널이 다시 visible 슬롯에 바인딩

### ROI / 녹화 / 부가 기능

- ROI 편집은 실제 채널 기준 유지
- 녹화 탭 카메라 선택은 기존 4채널 전체 기준 유지
- 녹화 탭 프리뷰는 기존처럼 별도 카메라 선택 콤보를 따름

## 영향 범위

- CCTV 메인 라이브 화면 레이아웃
- 채널 카드 선택 로직
- `MainWindowController`의 슬롯-채널 매핑 방식
- ROI 대상 위젯 탐색 방식
- Zone table 공급 데이터 기준

외부 설정 파일 형식이나 DB 스키마 변경은 없습니다.

## 확인 방법

1. 앱 실행 후 CCTV 메인 화면 진입
2. 채널 카드 1개만 선택했을 때 단일 대형 뷰로 표시되는지 확인
3. 두 번째 채널을 선택했을 때 2분할로 전환되는지 확인
4. 세 번째 채널을 선택했을 때 4분할로 전환되는지 확인
5. 선택된 카드를 다시 클릭했을 때 자연스럽게 해제되고 레이아웃이 줄어드는지 확인
6. 마지막 1개 선택 상태에서는 해제되지 않는지 확인
7. 비선택 채널도 썸네일과 상태 점이 계속 갱신되는지 확인
8. ROI 대상 변경 후 ROI 그리기/삭제가 해당 실제 채널에 적용되는지 확인
9. 녹화 탭 미리보기와 수동 녹화 대상 선택이 기존처럼 동작하는지 확인

## 리뷰 포인트

- 슬롯 재배치 이후 `CameraChannelRuntime`의 bind/unbind 흐름이 안전한지
- 선택 집합 변경 시 ROI 대상 채널과 화면 표시 채널이 일관되게 유지되는지
- 1/2/4 전환 과정에서 FPS 표기, ROI 오버레이, ReID 테이블이 깨지지 않는지
- 비선택 채널을 백그라운드 유지하는 현재 정책이 성능 측면에서 적절한지

## 참고

- 이번 변경은 레이아웃/UI 개선이 목적이며, 스트림 최적화 정책 변경은 포함하지 않습니다.
- 실제 스트림 수를 선택된 채널만큼만 줄이는 최적화는 후속 작업으로 분리하는 것이 안전합니다.
